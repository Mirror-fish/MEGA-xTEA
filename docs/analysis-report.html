SECTION 03 — 核心发现

MEGAnE 速度优势的代码根源

MEGAnE 处理 30x WGS 约 1 小时，xTea 需 8-12 小时。速度差异来源于四个层面。

3.1

C++ 加速模块

MEGAnE 将 BAM 处理的核心逻辑编译为 C++ 本地代码，使用 g++ -pthread -O2 -lhts 编译优化。

extract_discordant.cpp ~1,250行
  • htslib 线程池 + mutex 保护的 I/O 缓冲池
  • 每线程分配 1 GB I/O 缓冲区
  • 一次扫描同时提取: 嵌合读段 + 不一致配对 + 缺失断点
  • SA/XA tag 直接解析嵌合比对，无需重比对
MEGAnE/cpp/extract_discordant.cpp:80-196
extract_unmapped.cpp ~300行
  • DNA → 2-bit 编码（64位表示一个32-mer）
  • K-mer 过滤替代完整序列比对
  • 排序 k-mer 向量 + 二分查找 = O(log N)
  • 仅输出含重复序列 k-mer 的未比对读段
MEGAnE/cpp/extract_unmapped.cpp:52-106
关键洞察: xTea 全部使用 Python 处理 BAM，通过 pysam 包装 htslib 调用，存在 Python-C 接口开销。MEGAnE 直接操作 htslib C API，消除中间层。
3.2

K-mer 策略 vs 重比对策略

维度 MEGAnE xTea
候选筛选11-nt k-mer 索引重复序列 → 快速过滤每候选位点调用 BWA MEM 重比对
BWA 调用次数0 次3-4 次/位点
BWA 种子频率N/ABWA_SEED_FREQ=10 (默认500)

  • C++ 层: 线程池处理 BAM + mutex I/O

SECTION 05 — 最关键发现

SVA 检测表现差异的代码根源

SVA (SINE-VNTR-Alu) 是最具挑战性的 TE 类型：其内部 VNTR 区域导致拷贝长度高度可变，传统通用过滤策略极易产生假阴性。

核心结论: xTea 之所以在 SVA 检测上表现更好，根本原因是它实现了一套 VNTR-aware 的专用检测框架，而 MEGAnE 将 SVA 作为通用 SINE 处理，缺乏对 SVA 三段式结构的感知。

5.1 VNTR 区域处理 — 关键差异

SVA 的 VNTR（可变数目串联重复）区域会导致不同个体间的拷贝长度自然变异，这会破坏基于一致性的过滤逻辑。

xTea — VNTR 感知
# x_clip_disc_filter.py:1352-1353
# 当两侧一致性检查都失败时:
if bl_consist == False \
   and br_consist == False \
   and global_values.IS_CALL_SVA == False:
    continue # 仅非SVA时跳过
# SVA 模式: 允许通过！
SVA 的 VNTR 变异会导致两侧都不一致 — xTea 理解这一点并放宽检查
MEGAnE — 通用处理
# 无 VNTR 特殊处理
# pair_breakpoints.py:119
if abs(L - R) < 50: # 统一 TSD 距离
    paired = True
# 所有 TE 类型使用相同阈值
# 不区分 SVA 的 VNTR 变异
统一的一致性检查会将 VNTR 导致的合法变异误判为假阳性

5.2 SVA 专用注释扩展

# xTea/xtea/x_TEI_locator.py:1095-1096

    boundary_extnd = global_values.SVA_ANNOTATION_EXTND # 200bp 扩展

原因: RepeatMasker 对 SVA 的注释边界准确度低于 LINE1/Alu，因为 SVA 的 VNTR 区域变异使精确边界难以确定。xTea 通过扩展 200bp 补偿这一不足。

7.2 技术融合路线图

P1
Phase 1: C++ 高速框架底座
以 MEGAnE 的 C++ BAM 处理模块为核心，重构 extract_discordant.cpp 和 extract_unmapped.cpp。保留 k-mer 过滤、多线程 I/O、Absent ME 检测。建立 Python-C++ 绑定接口 (pybind11 或 ctypes)。
P2
Phase 2: 移植 SVA 专项优化
将 xTea 的 IS_CALL_SVA 逻辑体系移植: VNTR 一致性放宽、200bp 注释扩展、结构感知参数 (CNS_HEAD/POLYA_START)、post_processing_SVA() 过滤层级。同时为其他 TE 类型建立类似的专用过滤框架。
P3
Phase 3: 集成 ML 基因分型
采用 xTea 的 15 维特征提取 + Deep Forest 分类器，替换 MEGAnE 的高斯拟合。需重新训练模型以适应新框架的特征分布。考虑使用 ONNX Runtime 进行推理加速。
P4
Phase 4: 长读段支持
参考 xTea 的长读段模块，添加 PacBio/Nanopore 数据处理能力。长读段可直接跨越完整 TE 插入，大幅提升断点精度和 SVA/LINE1 全长检测率。
P5
Phase 5: 统一输出与联合分析
统一 VCF 输出格式 (兼容 MEGAnE 的多等位处理和 xTea 的详细 SVTYPE 注释)。整合 MEGAnE 的分块联合分析框架，支持 >10,000 样本群体分析。

7.3 预期技术挑战

架构层面
  • C++ 与 Python 的高效绑定接口设计 (pybind11 vs ctypes vs Cython)
  • 不同过滤策略的参数空间统一
  • MEGAnE 的覆盖度自适应阈值与 xTea 的固定阈值的协调
算法层面
  • ML 基因分型模型的重训练（新框架特征分布不同）
